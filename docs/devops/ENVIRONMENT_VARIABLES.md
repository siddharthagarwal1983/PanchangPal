# Environment Variables — Canonical Inventory (PanchangPal)

Version: 1.0.0
Last Updated: 2026-07-12

Purpose: the single source of truth for **every environment variable actually referenced in the
repository**. Generated by scanning for `process.env.*`, Edge Function `readEnv`/`get(...)`,
`EXPO_PUBLIC_*`, and GitHub Actions `secrets.*`. **Nothing here is invented** — each row cites where
it is read.

> Method: `grep` sweep over `*.ts/tsx/js/mjs/yml/sh` (excluding `node_modules`) on 2026-07-12.
> If you add a new variable, add it here and to the matching `.env.*.example` + `SECRETS_MATRIX.md`.

Legend — **Scope**: `client` (baked into the mobile bundle, public), `server` (Edge Function
secret), `ci` (GitHub Actions), `local` (developer/tooling only). **Sensitive?**: Yes = must never
be committed or exposed to the client.

---

## 1. Client / mobile (public — `EXPO_PUBLIC_*`)

These are read by `apps/mobile/app.config.ts` and surfaced via `expo-constants` to
`apps/mobile/src/data/supabaseClient.ts`. They are **public by design** (embedded in the app
binary); no server secret is ever an `EXPO_PUBLIC_*` value (ADR-030).

| Name | Purpose | Example format | Required? | Used by | Dev | Staging | Prod | CI | Sensitive? | Default | Owner |
|---|---|---|---|---|---|---|---|---|---|---|---|
| `EXPO_PUBLIC_SUPABASE_URL` | Supabase project URL the app calls | `https://abcd.supabase.co` | Yes | `app.config.ts`, `supabaseClient.ts` | ✓ | ✓ | ✓ | build | No (public) | — | Platform |
| `EXPO_PUBLIC_SUPABASE_ANON_KEY` | Public anon key (RLS is the boundary) | `eyJhbGciOi...` (JWT) | Yes | `app.config.ts`, `supabaseClient.ts` | ✓ | ✓ | ✓ | build | No (public, but rotate on abuse) | — | Platform |
| `EXPO_PUBLIC_REVENUECAT_KEY` | RevenueCat public SDK key (client) | `appl_xxx` / `goog_xxx` | Optional (F-4 subscription) | `app.config.ts` | ✓ | ✓ | ✓ | build | No (public SDK key) | — | Platform |

> Note: `SUPABASE_FUNCTIONS_URL` appears only in a **comment** in
> `apps/mobile/src/domain/guru/ProductionGuruTransport.ts` (`\`${SUPABASE_FUNCTIONS_URL}/ask-guru\``)

## 5. Deliberately absent (searched, **not present** — do NOT invent)

The audit searched for these and found **no references**. Do not add them to templates or code
without a real usage site:

- `APPLE_*` / `ASC_*` (App Store Connect API key/issuer) — **not** env vars in this repo.
- `GOOGLE_*` / Android service-account JSON — **not** env vars in this repo.
- `STRIPE_*` — not used (payments are RevenueCat, not Stripe).
- `EAS_*` beyond `EXPO_ACCESS_TOKEN` — no `eas.json` exists yet.

**Store credentials are EAS-managed, not env vars.** Apple/Google signing and submission
credentials are expected to live in the **EAS credentials service** (and App Store Connect / Play
Console), configured when `eas.json` and the `eas-build`/`submit` steps are made real (Beta Readiness
milestone). They are documented as a process in `docs/SETUP.md`, not as environment variables here.

## Change control

When you introduce a variable: (1) reference it in code; (2) add a row here with its usage site;
(3) add a placeholder to the correct `.env.*.example`; (4) classify it in `SECRETS_MATRIX.md`;
(5) if a deploy needs it, add it to `scripts/preflight.sh`.
